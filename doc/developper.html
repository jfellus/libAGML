libAGML User's Documentation

What is libAGML ?

libAGML is a software library for Asynchronous Decentralized Machine Learning. It allows a set of computers in a network to run optimization algorithms and statistical learning chains in a fully decentralized and asynchronous fashion. In the AGML paradigm, decentralization and asynchrony have the following definitions:

  • Decentralization : All computation units play the same role. That is, there isn't any /master node/ or specific required connectivity (obviously appart from having a strongly connected network). In AGML, groups of nodes are functionally homogeneous, meaning that all nodes in a given group execute the very same algorithm, and communicate with its direct neighbors without any distinction. In practice, when two node groups running different algorithms need to exchange data, output nodes are never aware of which input node they are talking to. They only know that they are sending data to one of the nodes in the destination group. Consequently, data exchanges between algorithms is always conceived at the NodeGroup level, never at the individual Node level.

  • Asynchrony : AGML is mostly built on asynchronous communications. This means that when some node sends data to a neighbor, it is not allowed (or at least discouraged) to wait for the neighbor's response. The advantage of enforcing such constraint appears when receiver nodes are busy or require a long running operation to process incoming data before answering to the sender. Any algorithm or protocol that is built on synchronous communication, where senders wait for receiver response before doing any further computation, is likely to encounter dead-locks or catastrophic slow-downs when fast processing of incoming data and nodes availability is not always formally guaranteed. This is actually the case in most optimization algorithm in machine learning where updating parameters both locally and in reaction to incoming data can not be carried out simultaneously, thus requiring synchronisation barriers between long-running operations. Asynchronous communications circumvent this drawback, but generally require dedicated optimization schemes, thus tagged as /asynchronous/.

It is written in C++ and is (initially) targetted to POSIX platforms.
