agml_gui : Live execution monitoring GUI

agml_gui is a Graphical User Interface for monitoring and controlling the execution of a model. It is based on Node Webkit and thus requires that Nodejs and Webkit be installed (most Linux distros provide packages for them).

agml_gui displays in a single window every remote Hosts currently involved in AGML computation and report runtime information about the corresponding NodeGroups and individual Nodes (iteration, emission and reception speeds).

Connection to running daemons

For agml_gui to discover running daemons, one or more addresses must be entered in the AGML Infos providers field in the left panel. To declare a daemon, the format is as follows:

agmlinfo://ip:port

port defaults to 10001 if omitted. Once added, information from every daemon connected to one of those specified in the AGML Infos providers field are automatically retrieved every second.

libAGML takes care of inter-daemon communications to provide agml_gui with information from all daemons. The user only needs to connect to one daemon address.

In order to enable transmission of Node information between agmld daemons (disabled by default to maximize computing performance), type in a shell: $ agml <deamon_ip:port> start_infos

Text entered in the various fields of agml_gui is persisted in the /home/myuser/.agml directory.
